Основные правила работы с гитом 
В мастер и дев ничего не класть, туда попадает код только пулреквестами из дочерних веток
Работа над каждой отдельной фичей, классом, багфиксом, над чем угодно - через дочернюю ветку дева
Например, как это будет работать с конфиг классом

1. git clone https://github.com/toradziro/webserv
2. git checkout dev
3. git pull
4. git checkout -b Dima/create_Config_parser_class
5. Создаем файлы, вносим правки
6. git pull origin dev
7. Переходим на сайт гита, переключемся на свою ветку, нажимаем create pull request
8. Создаем реквест к деву, если есть мердж конфликты решаем совместно, после этого тот, чьи правки, просит второго их просмотреть и оставить комметарии и правки, когда все комментарии решены - можно вливать ветку
